### Описание проекта

**Задача**: разработать/расширить структуру Data Lake. Создать витрины под бизнес-требования с автоматизацией обновления.

## Архитектура хранилища

Хранилище содержит следующие слои:

- **RAW** — сырые данные событий в исходном формате. Предоставляются из внешнего источника
- **ODS** (Operational Data Store) — предобработанные данные с партиционированием по дате
- **DDS** (Detail Data Store) — промежуточные обработанные данные (обогащенные координатами, городами)
- **DML** (Data Mart Layer) — готовые витрины для аналитиков

## Витрины данных (слой DML)

### UsersMart
Информация о пользователях (ID, город, координаты)

### ZonesMart
Статистика по городам (сообщения, реакции, подписки за неделю/месяц)

### FriendsMart
Рекомендации друзей на основе общих каналов, переписки и географической близости

## Формат хранения

- **Путь**: `/user/username/project7/dml/`
- **Формат**: **Parquet** (оптимизированный колоночный формат для больших данных)

### Преимущества Parquet

- Эффективное сжатие и хранение
- Сохранение схемы данных и типов колонок
- Высокая производительность при аналитических запросах
- Поддержка партиционирования по дате

## Компоненты системы

### Файлы
- `geo_classes.py` — классы для обработки данных
- `run_geo.py` — точка входа с параметром для выбора задачи

### Функции для расчета витрин
- `user_mart.py`
- `zones_mart.py`
- `friends_mart.py`

### Оркестрация
- **DAG Airflow** — оркестрация выполнения задач

## Тестирование

Тестирование проводилось с использованием:
- Расчета всех витрин через **DAG в Airflow**
- Отдельного расчета каждой витрины с помощью **Jupyter**
